## Descriptive Title

Access to OS-level install verification mechanism for marketplace-distributed apps

## Request Type

Interoperability Request

## Feature Name

iOS System-Level Install Verification and Attestation Capability (cryptographic install
authenticity mechanism)

## Reason for Request

iOS includes a system-level capability that generates cryptographically signed attestation for
app installations. This attestation serves as verifiable proof that an app was legitimately
installed through a trusted distribution channel and is relied upon by developers and
measurement partners to detect fraud and validate installs.
This request does not seek access to raw security signals or internal fraud detection systems.
It seeks access to the same OS-level install attestation capability that iOS already provides to
the App Store.
Apps distributed via the App Store benefit from OS-level install verification that is
independently verifiable and trusted across the ecosystem, whereas apps distributed via
alternative marketplaces cannot provide equivalent proof of authenticity.
The relevant functionality is therefore the OS-level generation of cryptographically verifiable
install attestation and its provision to apps and developers, which constitutes a system-level
capability controlled by iOS and currently available to the App Store and its users.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the OS-level install verification to provide developers with install authenticity
equivalent to App Store receipts:
1. Install fraud prevention: Developers and MMPs will be able to validate cryptographically
signed receipts to distinguish legitimate Onside installs from fraudulent installs, bot-generated
events, and device farm attacks — equivalent to how App Store receipts are used today.
2. Marketing attribution trust: Developers will be able to provide verification receipts to
attribution partners, enabling accurate marketing measurement for apps distributed via Onside
— equivalent to the trust level that App Store receipts provide.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

There are no viable alternatives. The install verification mechanism relies on two things that
only Apple controls:
1. OS-level receipt generation: App Store receipts are generated by iOS at the operating
system level using Apple's private cryptographic keys.The install verification mechanism relies
on OS-level cryptographic attestation that can only be generated and signed within iOS. Third
parties cannot replicate this functionality outside the operating system, as they do not have
access to system-level attestation mechanisms.
2. Even if alternative marketplaces attempt to generate their own verification tokens, such
tokens would not carry independent trust, as they are not issued by the operating system and
cannot be cryptographically verified against a system-controlled authority.
Only Apple can extend the OS-level install verification feature to apps distributed through
alternative marketplaces.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- Install attestation data will contain only minimal installation metadata (app ID, install time,
version). No individual user identifiers or personal data will be included.
- All access would be subject to iOS entitlements, audit logging, and EU GDPR requirements.
Technical Details:
- iOS already generates cryptographically signed install receipts for all App Store installations.
The infrastructure exists and is operational.
- Apple can extend access to the same receipt generation for apps distributed through
authorized alternative marketplaces, delivered via API.
Success Criteria:
- Onside-distributed apps receive cryptographically signed install verification receipts with
security properties equivalent to App Store receipts.
- Developers and MMPs can validate receipts using Apple-provided public keys.
- Receipt generation occurs automatically at OS level without requiring per-app SDK
integration.

May 4, 2026, from Apple to Developer:
We have reviewed your submission and determined that effective interoperability exists.
Specifically, developers and their chosen partners have access to Apple's App Attest which
allows for the cryptographic verification of apps that were code-signed by Apple, using SEP-
bound keys with certificates issued by Apple's public key infrastructure. App Attest also
provides a fraud metric capability that relying parties can use to detect inauthentic app installs.
See the developer documentation available at https://developer.apple.com/documentation/
devicecheck/establishing-your-app-s-integrity.
For this reason, we are closing your request and not moving it to the next phase of our
interoperability process.
