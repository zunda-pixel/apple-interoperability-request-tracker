## Descriptive Title

Access to the app installation API without the additional confirmation prompt

## Request Type

Interoperability Request

## Feature Name

iOS App Installation Confirmation Flow (system-level install authorization mechanism used by
the App Store)

## Reason for Request

Onside is an alternative app marketplace for iOS operating within the EU under the DMA.
When users install apps via the App Store, they tap “Get” and installation proceeds
immediately without any additional system-level confirmation step.
By contrast, when users install apps via Onside (or any third-party marketplace using
MarketplaceKit), iOS introduces an additional system confirmation sheet that requires the user
to confirm the installation again.
This results in differential treatment of identical user intent:
- In both cases, the user explicitly initiates installation
- Only third-party marketplaces are subject to an additional confirmation step
The relevant functionality is therefore not “installation” itself, but the ability to initiate installation
without an additional confirmation prompt.
This functionality is clearly available to Apple’s own App Store and constitutes a system-level
capability controlled by iOS.
Under Article 6(7) DMA, Apple must provide access to the same software features and
functionalities that are available to its own services.
The additional confirmation step does not introduce new security or consent:
- Apps remain subject to notarization, code signing, and sandboxing
- User intent is already explicitly expressed via the initial “Get” action
Therefore, the additional confirmation represents pure UX friction imposed only on competing
marketplaces, creating a material competitive disadvantage.
MarketplaceKit is not an effective alternative, as it is the mechanism through which this
asymmetric behavior is enforced.
Without access to the same installation flow, Onside cannot offer equivalent user experience or
compete on equal terms, contrary to the objectives of the DMA.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the iOS App Installation Confirmation Flow to provide users with a frictionless
app installation experience equivalent to the App Store. When a user taps "Get" (or equivalent)
on an app in Onside, installation will commence immediately without an additional iOS-level
confirmation dialog, matching the UX users receive via the App Store. This eliminates the
competitive disadvantage created by the additional MarketplaceKit confirmation prompt and
delivers genuine interoperability.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

We have evaluated the following alternatives:
1. MarketplaceKit (current requirement): iOS mandates third-party marketplaces use
MarketplaceKit, which introduces the additional confirmation sheet. This is not a genuine
alternative — it is the only available path for third-party distribution and explicitly includes the
friction we seek to eliminate.
2. Building our own installation confirmation suppression mechanism: Third-party developers
cannot suppress or modify iOS system-level dialogs. This is not technically feasible.
3. Web-based marketplace with native links: A web marketplace cannot execute native app
installations; users would still encounter the same MarketplaceKit confirmation flow or would
need to route through the App Store (negating the interoperability objective).
4. Accepting the additional confirmation as competitive compromise: While we could continue
with the current flow, this perpetuates the exact interoperability gap the DMA is designed to
address. The gap is artificial — driven solely by Apple's control over iOS, not technical
necessity. Accepting it would mean Onside cannot offer equivalent user experience, defeating
the interoperability mandate.
None of these alternatives provide effective interoperability. Only direct access to the iOS App
Installation Confirmation Flow mechanism (used by App Store) can eliminate the friction
differential.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- The streamlined confirmation flow does not bypass security or consent mechanisms. It simply
eliminates a redundant iOS-level dialog that App Store does not trigger.
- Alternative marketplaces remain subject to existing iOS security policies (code signing,
notarization, entitlement restrictions).
- User consent for app installation is retained — users explicitly select "Get" in Onside just as
they do in the App Store.
- No personal data exposure is created by this feature access.
Technical Details:
- The iOS App Installation Confirmation Flow is already integrated into iOS for the App Store.
- Apple can provide equivalent integration for entitled alternative marketplaces through API or
entitlements.
- Precedent: Apple provides system integrations to App Store that competing services access
via APIs.
Success Criteria:
- Onside users experience the same installation friction level as App Store users (no additional
confirmation prompts beyond standard security policies).
- Metrics: Conversion rate from app selection to installation in Onside matches or approaches
App Store baseline, demonstrating equivalent UX competitiveness.
- No reduction in security or user consent compared to App Store flow.

## Communications with Developer

April 10, 2026, from Apple to Developer:
Thank you for your interoperability request.
It is currently in Phase I - Eligibility review. We will reach out via this forum if we need
clarification of your request.
If we do not reach out for clarification, we will endeavor to provide you with the outcome of the
initial assessment of our Phase I - Eligibility review within 20 working days of the date of your
request, which does not include any public holiday as defined by the European Commission.
May 4, 2026, from Apple to Developer:
We have reviewed your submission and determined that there is parity between the App Store
and alternative app marketplaces, such as Onside, in terms of access to the functionalities
described in your request. In particular, any app marketplace or the App Store set as the
default app for app installation on iOS has the confirmation step disabled automatically and
users can also manually disable the confirmation step in the alternative app marketplace's
settings.
For this reason, we are closing your request and not moving it to the next phase of our
interoperability process. This concludes our review under Article 6(7) DMA.
If you have additional questions, you may reach out to us through Apple Developer Forums
(https://developer.apple.com/forums/), Feedback Assistant (https://developer.apple.com/
feedback-assistant/), or a code-level support request (https://developer.apple.com/support/
technical/).
