## Descriptive Title

Access to OS-level crash reporting for marketplace-distributed apps

## Request Type

Interoperability Request

## Feature Name

iOS System-Level Crash Diagnostic Collection and Aggregation Capability

## Reason for Request

iOS includes a system-level capability that automatically detects crashes, collects diagnostic
information, and aggregates this data for installed apps. These aggregated diagnostics are
made available to developers through App Store Connect without requiring any SDK
integration.
This request does not seek access to raw crash dumps or device-level diagnostic logs. It seeks
access to the same aggregated crash diagnostics and reporting functionality that iOS already
provides to App Store developers.
Apps distributed via the App Store benefit from OS-level crash detection and aggregation
without requiring any developer action, whereas apps distributed via alternative marketplaces
must rely on optional SDK integrations that provide incomplete and inconsistent coverage.
Without access to these diagnostics, alternative marketplaces cannot monitor app stability or
ensure quality across their catalog, while Apple retains full visibility into apps distributed via the
App Store.
The relevant functionality is therefore the OS-level collection and aggregation of crash
diagnostics and their provision to developers, which constitutes a system-level capability
controlled by iOS.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the OS-level crash reporting to provide developers with crash diagnostics
through the Onside Developer Dashboard — equivalent to what App Store Connect provides.
Developers will be able to monitor app stability and identify critical issues for their apps
distributed via Onside.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

We have evaluated the following alternatives:
1. Third-party crash analytics SDKs (Firebase Crashlytics, Sentry, etc.): These require per-app
SDK integration by every developer. App Store developers get crash diagnostics without any
SDK — the OS collects data automatically. Requiring SDK integration for alternative
marketplace apps creates an asymmetric developer burden that does not exist for App Store
distribution. Additionally, even if developers integrate these SDKs, the data is sent to the
developer — not to the alternative marketplace. This means Onside has no visibility into crash
rates across its catalog, making it impossible to ensure the quality and stability of apps for end
users.
2. Accepting limited crash visibility: Developers could operate without crash diagnostics. This
perpetuates the interoperability gap: App Store developers have automatic crash reporting;
Onside developers lack equivalent tools.
None of these alternatives provide equivalent functionality to the OS-level crash reporting
available to App Store. Only access to the same system-level mechanism can eliminate the
developer disadvantage.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- Crash reports will be aggregated and anonymized. No individual user identifiers or personal
data will be provided to developers.
- Data scope covers only crash diagnostics — not in-app behavior or user activity.
- All access would be subject to iOS entitlements, audit logging, and EU GDPR requirements.
Technical Details:
- iOS already collects crash diagnostics for all installed apps and makes them available
through App Store Connect. The infrastructure exists and is operational.
- Apple can extend access to the same crash reporting for apps distributed through authorized
alternative marketplaces, delivered via API.
Success Criteria:
- Onside developers can access crash diagnostics without requiring per-app SDK integration.
- Crash reporting coverage matches what is available in App Store Connect.
- Data is available with equivalent freshness as App Store crash reporting.
