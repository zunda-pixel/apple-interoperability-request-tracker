## Descriptive Title

Access to the system-level app installation confirmation sheet metadata for alternative
marketplaces

## Request Type

Interoperability Request

## Feature Name

iOS System-Level App Installation Confirmation Sheet Metadata Resolution

## Reason for Request

When a user initiates app installation from an alternative marketplace on iOS, the system
displays a confirmation sheet that presents key app information, including app name,
developer identity, and icon. This information is resolved by iOS through a system-level
metadata lookup functionality as part of the installation security flow.
This metadata is already available to the operating system and is displayed directly to the user
at the point of installation.
However, alternative marketplaces do not have access to the same metadata resolution
capability. As a result, marketplaces must independently obtain or maintain app metadata,
which can lead to inconsistencies between marketplace listings and the information displayed
by the system during installation.
This creates a discrepancy where users may see different app identity information before and
during installation, undermining trust and consistency in the installation flow.
The relevant functionality is therefore the OS-level capability to resolve and provide
authoritative app identity metadata used in the installation confirmation process, which
constitutes a system-level feature controlled by iOS.
This request does not seek access to App Store-specific content or extended metadata. It only
seeks access to the same metadata that iOS already resolves and displays to users during the
installation confirmation flow.
Providing access to this capability would ensure that alternative marketplaces can present
consistent and verified app information aligned with what the operating system displays.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the system-level metadata resolution capability to ensure that app information
presented in the marketplace matches the information displayed by iOS during installation.
This enables:
- consistent app identity: app name, developer, and icon match system confirmation sheet
- user trust: users see the same information before and during installation
- reduced duplication: no need for separate metadata pipelines for system-level identity data

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

We have evaluated potential alternatives and concluded that no effective or equivalent
solutions exist.
Developer-provided or manually maintained metadata cannot guarantee consistency with the
information displayed by the operating system during installation.
Extracting metadata from app bundles does not provide the same verified information that iOS
resolves and presents in the confirmation sheet.
Only iOS has access to the authoritative metadata used during the installation process, and
therefore only system-level access can ensure consistency.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- This request does not involve access to user data.
- It only concerns application identity metadata that is already displayed to users by the
operating system during installation.
- No personal data, such as user activity is processed .
Technical Specification:
- iOS already resolves and displays app identity metadata during the installation confirmation
flow
- This capability is used internally by the system
- Apple can expose a read-only interface to authorized alternative marketplaces to access the
same metadata
- No additional data collection or processing is required
Success Criteria:
- Onside can query the metadata resolution API for any notarized application before and during
installation
- Metadata returned from the API matches what appears in the system installation confirmation
sheet
- Query latency is <100ms, suitable for real-time UI updates
- Metadata is verified and matches app bundle contents
- The API is documented and accessible through standard iOS frameworks
- Access can be restricted to DMA-compliant alternative marketplaces
- Apple may revoke access for applications that abuse the API or misrepresent metadata
