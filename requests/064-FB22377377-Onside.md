## Descriptive Title

Access to system-level in-app app promotion and installation views for alternative
marketplaces

## Request Type

Interoperability Request

## Feature Name

OS System-Level In-App App Promotion and Installation Presentation Capability

## Reason for Request

iOS provides system-level in-app presentation capabilities that allow an application to promote
and present another app for installation without forcing the user to leave the current app
context. Apple provides these capabilities for the App Store through StoreKit surfaces such as
SKOverlay and SKStoreProductViewController. These features let apps recommend another
app and show its App Store product page directly inside the app experience.
Alternative marketplaces have no equivalent capability for apps they distribute.
As a result, when an app wants to recommend or install an app distributed through an
alternative marketplace, it cannot present a native, system-level overlay or embedded product
view comparable to the App Store experience. Instead, developers must redirect users to web
pages, external browser flows, or marketplace apps, creating additional friction and
interrupting the user journey.
This creates a structural asymmetry in in-app distribution and promotion. The same use case
— recommending another app from within an existing app — can be handled through a
system-optimized, low-friction UI when the destination is the App Store, but not when the
destination is an alternative marketplace.
The relevant functionality is therefore the OS-level capability to present an app promotion
overlay or embedded product page and initiate installation from within an app, which
constitutes a system-level feature controlled by iOS.
This request does not seek access to App Store-specific content or privileged commerce
systems. It seeks access to an equivalent iOS system-level presentation and installation
capability for apps distributed via authorized alternative marketplaces, so that developers can
promote and install those apps from within other apps under equivalent conditions.
Without access to this iOS capability, alternative marketplaces are disadvantaged in a core app
discovery and conversion flow, while Apple’s own distribution channel benefits from native in-
app promotion and installation surfaces. Under Article 6(7) DMA, this capability should be
made available to alternative providers under equivalent conditions.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the system-level in-app app promotion and installation presentation capability
to allow apps and partners to recommend and install Onside-distributed apps without leaving
the current app context.
This enables:
- in-app app recommendation overlays: showing a system-level overlay for an app distributed
through Onside
- embedded product presentation: presenting an app detail page inside the current app,
comparable to App Store product presentation
- reduced friction: allowing users to continue the install flow without being redirected to
external websites or separate browser journeys
- better developer conversion: enabling partners and developers to promote Onside-distributed
apps using the same class of native in-app surfaces currently available for App Store apps

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

No alternatives exist.
Web redirects and browser-based flows interrupt the user journey and do not provide the same
native, in-app presentation and installation experience.
Custom in-app UI cannot replicate this functionality because app installation remains
controlled by iOS, and third-party apps do not have access to equivalent system-managed
promotion and installation surfaces.
Only iOS can provide a system-level in-app presentation capability with equivalent integration,
security, and user experience.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards
- This request does not involve access to user data.
- It concerns only the presentation of app information and invocation of an installation flow that
remains subject to existing system security and user confirmation requirements.
- No personal data such as behavioral information, is processed.
Technical Implementation
- iOS already provides system-level in-app app promotion and product presentation
capabilities for the App Store through StoreKit surfaces such as SKOverlay and
SKStoreProductViewController
- Apple can expose an equivalent presentation capability for apps distributed through
authorized alternative marketplaces
- Existing system confirmation, notarization, and security requirements would remain
unchanged
- No additional data collection or processing is required
Success Criteria
- Apps can present Onside-distributed apps using a native system-level overlay or embedded
product page from within another app
- Users can begin the installation flow for an Onside-distributed app without being forced into
an external browser-only journey
- The presentation and interaction model is equivalent in quality and integration to the App
Store’s in-app promotion surfaces
- The mechanism is documented and available to DMA-compliant alternative marketplaces
under equivalent conditions
- Existing system security checks and user confirmations remain preserved

May 4, 2026, from Apple to Developer:
We have reviewed your submission and determined that your request - seeking in-app
promotion and installation functionalities - does not concern a feature of iOS or iPadOS
accessed by the App Store, within the meaning of Article 6(7) DMA. For this reason, we are
closing your request and not moving it to the next phase of our interoperability process.
As this determination is based on a non-technical consideration (i.e., whether what is
requested is available to / used by Apple services or hardware), it is not eligible for the dispute
resolution mechanism.
This concludes our review under Article 6(7) DMA.
Without prejudice to the legal analysis under Article 6(7) DMA set out above, our Worldwide
Developer Relations team will be in touch shortly to discuss how we can support in-app
installation functionality for your app marketplace.
If you have additional questions, you may reach out to us through Apple Developer Forums
(https://developer.apple.com/forums/), Feedback Assistant (https://developer.apple.com/
