## Descriptive Title

Access to the MarketplaceKit app installation and update APIs without geographic restrictions

## Request Type

Interoperability Request

## Feature Name

iOS System-Level App Installation and Update Capability (without geographic restriction)

## Reason for Request

iOS includes a system-level capability that allows app stores to install and update apps on user
devices. This capability is used by the App Store without geographic restriction, allowing users
to install and update apps regardless of their location.
By contrast, equivalent APIs provided to alternative marketplaces through MarketplaceKit are
restricted based on the user’s geographic location. When a user travels outside the EU,
installation and update functionality is disabled for alternative marketplaces, while the App
Store continues to function normally on iOS.
This results in differential treatment of the same user on the same operating system and
device, depending solely on the distribution channel used.
This breaks continuity of service: apps installed via alternative marketplaces cannot be
updated while the user is traveling, creating security risks and degraded user experience.
The relevant functionality is therefore the OS-level capability to install and update apps without
geographic restriction, which constitutes a system-level feature controlled by iOS.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use unrestricted app installation and update APIs to:
1. Serve iOS EU users traveling abroad: EU users (identified by phone number or manual
country selection) who installed Onside and its apps will continue to receive updates and install
new apps when traveling outside the EU — the same way App Store works for those iOS users
today. Currently, if an EU user travels outside the EU, Onside stops functioning on iOS while
the App Store continues to work normally.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

There are no alternatives. The geographic restriction is enforced by Apple at the MarketplaceKit
API level. When a user is outside the EU, Apple's APIs reject installation and update requests.
Onside has no way to bypass or work around this — Apple controls the underlying system-
level installation functionality.
The only solution is for Apple to remove the geographic geofencing from MarketplaceKit APIs,
providing the same unrestricted access that the App Store has.
Any application-level workaround is not possible, as installation and update operations are
exclusively controlled by iOS system APIs.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- Removing geographic restrictions does not require sharing user location data with Onside or
developers.
- All existing iOS security checks, code signing, and notarization requirements continue to
apply regardless of region.
Technical Details:
- The App Store already operates without geographic restrictions on the same iOS installation
infrastructure. The current implementation of MarketplaceKit includes geographic restrictions
that limit installation and update functionality outside the EU.
- Removing it requires no new infrastructure — only removing the geographic check from
existing API calls.
Success Criteria:
- EU users (identified by phone number or manual country selection) can install and update
Onside apps regardless of their current geographic location, the same way App Store works.
- EU users traveling outside the EU continue to receive app updates and install new apps
normally.
