## Descriptive Title

Request for deterministic background execution for connected device scenarios such as
enabling cross-device copy/paste or recent photo syncing.

## Request Type

Interoperability Request

## Feature Name

Background execution for features like Universal Clipboard

## Reason for Request

To meet our user needs, Microsoft needs the ability to keep the user's devices up to date when
content changes on one or the other. For example, the ability to push clipboard or file data
from the Windows PC to the iOS device or sync new photos and videos from the iOS device to
the Windows PC.

To achieve this in a seamless way, Microsoft needs the ability for the PC to deterministically
gain background execution time on the phone to perform actions that may-or-may-not be
long-running.

## Product Description

Windows, a general-purpose personal computing platform used globally for productivity,
communication, and multi-tasking across consumer and enterprise scenarios.

## How the Product Will Use the Feature

Windows would use deterministic background execution and background refresh on iOS to
enable seamless, user-initiated cross-device scenarios that keep an iPhone and a Windows PC
in sync without requiring users to foreground an app or repeatedly take manual actions.
These scenarios are already supported by Apple within its own ecosystem – for example,
between iPhone and macOS – where system-level background execution enables continuous
features such as Universal Clipboard and access to recent photos and files across devices.
Windows is seeking the ability to leverage similar background refresh capabilities on iOS to
enable equivalent cross-device productivity scenarios for users who choose a Windows PC
alongside their iPhone.
For example, this capability would enable:
• Cross-device copy and paste – When a user copies content on their iPhone, background
execution would allow the connected Windows PC to be notified and receive that content in
near real time, so it is immediately available to paste when the user switches to their PC.
Similarly, content copied on the PC could be made available on the phone without requiring the
user to foreground an app or explicitly initiate each transfer.
• Access to recent phone files and photos on PC: Background execution would allow the
Windows PC to stay up to date with newly captured photos, videos, or recently accessed files
on the iPhone.
In all cases, background execution would be used only to support user-initiated continuity
scenarios and lightweight synchronization. The intent is to enable reliable, timely updates that
make cross-device interactions feel immediate and natural – consistent with the experience
Apple already provides across iPhone and Mac – while respecting system constraints, user
control, and privacy expectations.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

Yes.
• Background execution frameworks offered by iOS are focused on continuing foreground work
in the background or for operations that are infrequent or are not time-sensitive.
• There are no triggers available on iOS that will grant an app execution time when user
content, such as photos or files, changes. This forces a need for a polling approach to monitor
changes to content.
• There are no acceptable frameworks for an external device such as a PC to reliably “wake”
our app’s process with any reasonable frequency on iOS to transmit data to or from the device.
BluetoothLE background modes are the closest available approximation but the execution time
granted is limited and documentation is unclear about how frequently the channel can be
utilized without throttling or degradation of experience. For example, with Universal Clipboard,
changes may be frequent.

teams will assess whether we are able to provide an effective interoperability solution and, if

be feasible or appropriate to provide an interoperability solution, including if providing effective
interoperability would compromise the integrity of the operating system. We have also not yet
conducted an assessment of the impact this request would have on Apple’s intellectual
property rights. We will inform you should it not be feasible or appropriate to implement your
request. We expect to have an update for you on this phase by 26 June.
As for your request for recent file syncing, we have determined that effective interoperability
already exists. The feature you are requesting interoperability with is currently available to
developers in the same way as it is available to Apple services. Further information is available

FileProvider (https://developer.apple.com/documentation/FileProvider). Because effective
interoperability exists, we have made no determination on the application of Article 6(7) DMA to
