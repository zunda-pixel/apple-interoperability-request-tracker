## Descriptive Title

Apple Wallet being pre-defined as default contactless application

## Request Type

Interoperability Request

## Feature Name

Definition of default contactless application

## Reason for Request

For a new iPhone, without any customization by the user Apple Wallet is the default
contactless / wallet application. That means that any 3rd party payment application needs to
bring the customer to define a different default payment application. Even if customer has
never used Apple Wallet, it is the predefined app.

## Product Description

Santander is working on the development of a mobile payments functionality within the mobile
banking app. To be able to compete in equal conditions vs OS apps, it is required that no app
is pre-defined as default.

## How the Product Will Use the Feature

Apple Wallet should follow the same journey as any third party application to be selected as
default contactless application. It should not come predefined by default. So that customer
faces the same default app definition journey for all the apps: third parties and Apple native
apps
