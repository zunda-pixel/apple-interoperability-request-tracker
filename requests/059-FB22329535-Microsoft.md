## Descriptive Title

Request for interoperable cross-device clipboard access between iOS and Windows.

## Request Type

Interoperability Request

## Feature Name

Clipboard / Universal Clipboard (cross-device copy and paste)

## Reason for Request

Cross-device copy and paste is a highly requested productivity feature and most-valued cross-
device experience for users who work across multiple devices.
Apple supports cross-device copy and paste between iOS and macOS through first-party
system integration. However, third-party platforms cannot offer comparable experience with
their own devices due to restrictions on clipboard access on iOS.
Specifically, an app cannot reliably read the device’s clipboard unless:
• The app is actively running in the foreground, and
• The user explicitly grants clipboard access each time
These constraints prevent an app from seamlessly synchronizing clipboard content from an
iPhone to a PC or other connected device in the background. As a result, cross-device copy
and paste require explicit, repeated user action and cannot function as a continuous, system-
level experience comparable to what is available between iOS and macOS.
While cross-device copy and paste is already supported seamlessly across Apple’s own
platforms, third-party developers do not currently have access to equivalent system
capabilities. Expanding access to iOS clipboard functionality through documented and
supported interfaces would enable third parties to deliver more consistent and intuitive cross-
device experiences. Apple’s help is needed to make this possible, as the current limitations are
driven by platform behavior and available system interfaces rather than by technical constraints
that third-party developers can address independently.

## Product Description

Windows, a general-purpose personal computing platform used globally for productivity,
communication, and multi-tasking across consumer and enterprise scenarios.

## How the Product Will Use the Feature

Windows would use interoperable clipboard access to enable cross-device copy-paste
between an iPhone and Windows PC.
This would allow users to:
• Copy text or other supported content on their iPhone and paste it directly on their Windows
PC, and vice versa.
• Perform common productivity workflows without needing to foreground an app or take
explicit repeated actions to initiate a transfer
• Experience clipboard synchronization as a lightweight, continuous capability rather than a
manual, app-driven operation
The intent is not to replace or replicate Apple’s system experience, but to enable a basic level
of cross-device productivity that users already expect when working across devices.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

Yes. Current iOS clipboard behavior requires an app to be in the foreground and requires user
confirmation for clipboard access, which makes background synchronization infeasible. These
constraints prevent third-party apps from implementing seamless, system-like clipboard
syncing across devices.
There are no alternative Apple-provided frameworks or documented APIs that allow an app to
be granted persistent clipboard access for the purpose of synchronizing clipboard content to a
connected device without repeated user interaction.

## Confidential Treatment

Fully available to other developers

## Additional Information

Cross-device copy and paste is a foundational productivity interaction that users expect to
“just work” across their primary devices. Apple already enables this experience across its own
platforms through system-level integration between iOS and macOS.
We request a mechanism by which an app can be granted persistent access to the device’s
clipboard for the purpose of initiating clipboard transfer to a connected device without
requiring the app to be foregrounded or the user to repeatedly grant access. Enabling this
would allow third-party platforms to offer a more consistent and usable cross-device clipboard
experience, while still operating within clearly defined system and privacy boundaries.
