## Descriptive Title

Default contactless applications if only one application is enabled

## Request Type

Interoperability Request

## Feature Name

Selection of default contactless application

## Reason for Request

User can define which is the default contactless application at the device. For that, user
accesses a settings screen where all Contactless applications are listed so that user can select
the desired one. In that settings screen, all apps that have contactless functionalities, are
included. Even if they are not in position to perform any contactless action. For example, Apple
Wallet will be on that list even if there are no cards provisioned and it would be inactive for any
contactless payment.

## Product Description

Santander is developing contactless payment functionality at iOS. If customer has cards
provisioned only at Santander, it doesn't make sense to have other apps defined as default as
it means a harder journey for the customer to start using it.

## How the Product Will Use the Feature

Applications without payment / contactless credentials activated should not be in the list of
contactless applications to be selected as default application. In case only one application has
contactless credentials ready to use, that should be the only application on that list, and it
should be the default application without the need of the user performing any action.
