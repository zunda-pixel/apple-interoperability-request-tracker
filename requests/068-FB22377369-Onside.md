## Descriptive Title

Access to the system-level eligibility and installation status API for alternative marketplace
installation in browsers

## Request Type

Interoperability Request

## Feature Name

iOS System-Level Marketplace Installation Eligibility and Status Feedback Capability

## Reason for Request

When a user initiates marketplace installation from the web on iOS, the operating system
manages the entire process, including eligibility checks, installation execution, and final
outcome.
However, web-based distribution channels have no programmatic way to:
- determine whether installation is possible for a given user
- receive feedback on whether installation succeeded or failed
- provide meaningful error handling or next steps
As a result, websites can only trigger the installation flow without any visibility into its outcome.
The same user action (clicking an installation link) produces no feedback on the initiating
context.
The relevant functionality is therefore the OS-level capability to provide eligibility and outcome
feedback for marketplace installation flows initiated from the web, which constitutes a system-
level feature controlled by iOS.
Without this capability, alternative marketplaces cannot provide reliable user experience or
handle errors appropriately, while system-integrated contexts can manage installation flows
with full awareness of system state.
Providing only a fire-and-forget installation capability without feedback constitutes a degraded
implementation and does not meet the requirement of effective interoperability under Article
6(7) DMA.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the system-level eligibility and status feedback capability to improve the user
experience of marketplace installation from web-based distribution channels.
This enables:
- eligibility awareness: determining whether installation is possible before initiating the flow
- status feedback: informing users whether installation succeeded or failed
- error handling: displaying clear and actionable messages when installation is not available
- flow continuity: guiding users to next steps after installation

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

No alternatives exist.
Web-based applications cannot access installation state or outcome, as this is controlled
exclusively by iOS system APIs.
Workarounds such as polling, custom URL schemes, or client-side logic cannot provide reliable
or real-time feedback, as they do not have access to system-level installation state.
Only iOS can provide accurate eligibility and status feedback for installation flows.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- This request does not involve access to user data.
- It only concerns eligibility and feedback on the outcome of an installation action initiated by
the user.
- No personal data, such as device identifiers and behavioral information is processed .
Technical Specification:
- iOS already determines eligibility and manages installation state internally
- This information can be exposed through a controlled interface to authorized contexts
- The same security and user confirmation flows remain unchanged
- No additional data collection is required
Success Criteria:
- Web-based contexts can determine whether marketplace installation is available before
initiating the installation flow
- Installation links provide consistent, actionable feedback across all browsers and applications
- Websites receive clear outcome signals indicating whether installation succeeded, failed, or
was cancelled
- Users are presented with understandable error messages when installation is not possible
(e.g., unsupported region, incompatible OS version, already installed)
- Feedback is provided in a timely manner suitable for real-time user interaction
- The feature is documented and available to DMA-compliant alternative marketplaces under
equivalent conditions
- The installation flow remains secure, with all existing system-level user confirmations
preserved
- The implementation is auditable and transparent to regulators

May 4, 2026, from Apple to Developer:
We have reviewed your submission and determined that your request falls outside the scope of
Article 6(7) DMA. In particular, your request seeks to ask Apple to create a new functionality on
iOS that would support the provision of eligibility and outcome feedback for alternative
marketplace installation flows from web browsers, which is not supported by Safari and not
available in the context of installation of the App Store on iOS. For these reasons, we are
closing your request and not moving it to the next phase of our interoperability process.
As this determination is based on non-technical considerations (i.e., whether Apple is legally
obligated to create new features within the meaning of Article 6(7) DMA that would not be used
by the requesting developer), it is not eligible for the dispute resolution mechanism.
This concludes our review under Article 6(7) DMA.
