## Descriptive Title

Information about being or not default NFC application

## Request Type

Interoperability Request

## Feature Name

Information on default NFC application

## Reason for Request

As part of DMA commitments, Apple allows third party applications to access iPhone NFC
capabilities and to be selected by the customer as default contactless application. Meaning the
application that will be invoked when customer taps into a POS or when customer double
clicks on the side button.
But default contactless application is not informed by the OS if the customer has or not
selected it as default. Therefore application can’t inform customer about “how to pay”.

## Product Description

Santander is launching NFC payments functionality within the mobile banking app, that
requires to know if Santander is or not the default NFC application.

## How the Product Will Use the Feature

iOS should inform application whether customer has selected it or not as default contactless
application. That way we can (or can't) tell the customer "double click on the side button" to
make a payment. Important for onboarding and payment processes, as well as customer
lifecycle management.
Right now, as example, for every card onboarding that we have, we need to redirect customer
to the definition of default contactless application. And might be the case that we already are,
but we do not know.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

In other OS that feature is already available. iOS does not offer any solution.
