## Descriptive Title

Access to the iOS default app store registry, preinstallation pipeline, and Home Screen default
placement

## Request Type

Interoperability Request

## Feature Name

iOS Device Setup Assistant App Store Configuration, Preinstallation Pipeline, and Home
Screen Default Placement Engine

## Reason for Request

Onside is an alternative app marketplace for iOS operating within the EU under the DMA.
A fundamental source of the App Store’s competitive advantage is its exclusive access to
system-level distribution mechanisms in iOS. These include:
- designation as the default app distribution channel,
- inclusion in the device preinstallation pipeline, and
- integration into the Device Setup Assistant, which is presented to all users during device
initialization.
These mechanisms are not merely product design choices; they are system-level capabilities
that determine how users first access software distribution channels on iOS devices.
Currently, these capabilities are exclusively available to Apple’s own App Store:
- The App Store is preinstalled on all devices
- It is placed by default on the Home Screen
- It is implicitly designated as the default distribution channel
- The Device Setup Assistant does not present any alternative marketplaces or allow users to
choose a preferred distribution channel
As a result, users are systematically not exposed to the existence of alternative marketplaces,
creating a structural barrier to competition that is independent of product quality or user
preference.
The relevant functionality is therefore the ability to be surfaced, designated, and selected as a
distribution channel at the system level during device initialization and early device use.
Under Article 6(7) DMA, Apple is required to provide access to the same operating system
features and capabilities that are available to its own services.
These include:
- system-level default designation mechanisms
- preinstallation and initial placement capabilities
- first-run user choice interfaces within the Device Setup Assistant
Denying access to these capabilities results in a structural discoverability disadvantage,
preventing alternative marketplaces from competing on equal terms.
This request does not seek promotional treatment or preferential placement.
It seeks equal access to system-level distribution mechanisms, enabling users to make an
informed choice between available marketplaces.
A non-skippable marketplace choice screen during device setup, presenting all eligible
marketplaces in randomized order, is a technically feasible and proportionate way to ensure
such interoperability.
Without access to these system-level capabilities, effective interoperability cannot be achieved,
as iOS itself prevents users from discovering alternative distribution channels.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the iOS Device Setup Assistant marketplace choice screen to ensure users are
presented with marketplace options during initial device configuration.
During device setup (the Device Setup Assistant flow), Onside will be presented to users in a
non-skippable choice interface alongside other entitled alternative marketplaces, presented in
randomized order. This enables users to make an informed marketplace selection at device
initialization, providing Onside with fair discoverability equivalent to App Store.
The choice screen will appear for:
- All new device activations in the EU
- Existing device settings workflows for users who have not yet selected a marketplace
preference
This mechanism will enable users to discover Onside as an equal alternative to App Store at
the moment they first interact with distribution channels, eliminating the discovery
disadvantage created by App Store's exclusive integration into device setup.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

We have evaluated the following alternatives:
1. Web-based distribution (current method): Alternative marketplaces can only be distributed
via the web — they are not available in the App Store. Users must independently discover the
marketplace's website, navigate to it in Safari, and complete a multi-step installation process.
This requires users to already know the marketplace exists, which is the core problem: there is
no system-level mechanism to inform users that alternative distribution channels are available.
2. Marketing and user acquisition campaigns: Marketing cannot replicate system-level
distribution, because it depends on prior user awareness, whereas OS-level mechanisms
define initial user awareness.
3. Accepting the current structure: Accepting this would mean Onside cannot compete on
product merits, as iOS architecture prevents users from even learning alternative options exist.
This perpetuates the interoperability gap the DMA is designed to address.
None of these alternatives provide system-level discovery equivalent to App Store's Device
Setup Assistant integration. Only device-setup choice screen access can enable fair user
awareness of alternatives from device initialization.

Fully available to other developers

## Additional Information

Privacy Safeguards:
- The choice screen mechanism requires no access to user personal data. It is a UI
presentation feature shown during setup before user account creation.
- Preinstallation requires no elevation of Onside's permissions beyond those currently available
to App Store as a native iOS application.
- All alternative marketplaces presented via choice screen will be subject to the same EU DMA
attestation and security review as App Store.
- Users retain full control: they can delete any preinstalled marketplace app or select no
marketplace during setup.
Technical Details:
- The iOS Device Setup Assistant is already engineered to present configuration choices
(language, privacy settings, iCloud, etc.). Adding a marketplace choice screen is a
straightforward extension of existing UX patterns.
- The preinstallation pipeline is already used for App Store. Extending it to other entitled
marketplaces follows the same technical pattern.
Home Screen placement can be managed through OS configuration (like other default Home
Screen apps on iPad).
- The capability to randomize order of presented options is standard iOS functionality.
Success Criteria:
- 100% of new device activations in the EU present the marketplace choice screen during
Device Setup Assistant.
- Users presented with the choice screen can select from Onside and other entitled alternative
marketplaces in randomized order.
- Choice screen presentation occurs consistently for existing devices when users access
marketplace settings, ensuring fair visibility over time.
- No reduction in security or user consent mechanisms compared to current App Store setup
integration.
