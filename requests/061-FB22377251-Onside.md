## Descriptive Title

Access to the background app update daemon without the EU-exit expiry

## Request Type

Interoperability Request

## Feature Name

iOS System-Level Background App Update Capability (without geographic or time-based
restriction)

## Reason for Request

iOS includes a system-level capability that automatically updates apps in the background
without user intervention. This capability ensures that apps remain up to date with the latest
features, bug fixes, and security patches.
For apps distributed via the App Store, this functionality operates without geographic or time-
based restriction. Apps continue to receive background updates regardless of the user’s
location.
By contrast, for apps distributed via alternative marketplaces, the current implementation
introduces a time-based restriction: when a user travels outside the EU, background updates
stop after a limited period. The same user, on the same operating system and device,
continues to receive updates for App Store apps.
This results in differential treatment of identical users based solely on the distribution channel
used.
This creates a direct security risk, as apps distributed via alternative marketplaces may remain
outdated and miss critical updates while App Store apps continue to be updated.
The relevant functionality is therefore the OS-level background app update capability without
geographic or time-based restriction, which constitutes a system-level feature controlled by
iOS.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the unrestricted background update daemon to:
1. Keep EU users' apps up to date abroad: Users who have installed Onside and its apps
within the EU will continue to receive automatic background updates when traveling outside
the EU — the same way App Store works for those users today. Currently, after ~30 days
outside the EU, Onside apps stop receiving background updates while App Store apps
continue updating normally on iOS.

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

There are no alternatives. The background update daemon is an OS-level process controlled
entirely by Apple. The expiry timer is enforced by iOS itself — Onside has no way to bypass or
extend it. The only solution is for Apple to remove the EU-exit expiry from the background
update daemon for alternative marketplace apps, providing the same unrestricted access that
the App Store has.
No application-level workaround exists, as background updates are executed exclusively by
system-level processes controlled by iOS.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- Removing the expiry does not require sharing user location data with Onside or developers.
- All existing iOS security checks, code signing, and notarization requirements continue to
apply regardless of region.
Technical Details:
- The App Store already uses the same background update daemon without any geographic or
time-based restrictions. The expiry timer is an additional gate Apple added specifically for
alternative marketplaces.
- Removing it requires no new infrastructure — only removing the expiry check from the
existing daemon.
Success Criteria:
- EU users (identified by phone number or manual country selection) receive automatic
background updates for Onside apps regardless of their current geographic location, the same
way App Store works.
- EU users traveling outside the EU continue to receive background updates indefinitely, with
no expiry timer.
