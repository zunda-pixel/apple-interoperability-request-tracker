## Descriptive Title

Access to the alternative marketplace installation flow for all WebKit-based browsers and apps

## Request Type

Interoperability Request

## Feature Name

iOS System-Level Marketplace Installation Invocation Capability (browser-independent)

## Reason for Request

iOS provides a system-level capability that allows users to initiate installation of alternative
marketplaces from web content. When such links are opened in Safari or system applications,
the operating system recognizes them and invokes the installation flow directly.
Apple provides an API that allows other browsers to implement support for this installation
flow. However, this requires each browser developer to independently adopt and integrate the
API.
As a result, the same installation link behaves inconsistently across browsers: it works
seamlessly in Safari and system applications, but depends on whether a given browser has
implemented the required API.
This creates fragmentation and places the burden of enabling a core system capability on
third-party browser developers, rather than exposing it as a uniform operating system feature.
The relevant functionality is therefore the OS-level capability to invoke marketplace installation
flows from web content in a browser-independent manner, which constitutes a system-level
feature controlled by iOS.
Requiring per-browser implementation does not provide equivalent access, as it results in
inconsistent behavior, delayed adoption, and reduced availability across distribution channels.
Under Article 6(7) DMA, this capability must be made available under equivalent conditions,
without dependence on third-party implementation, ensuring consistent behavior across all
browsers and applications relying on iOS to reach end users.

## Product Description

Onside — an alternative app marketplace for iOS that enables developers to distribute their
apps outside the App Store within the EU.

## How the Product Will Use the Feature

Onside will use the system-level installation invocation capability to ensure that marketplace
installation links behave consistently across all browsers and applications.
This enables:
- consistent installation experience across Safari and non-Safari browsers
- seamless installation from web links, email, messaging apps, and in-app web views
- elimination of browser-specific workarounds and redirects
- reduced user friction by removing the need to switch to Safari

## Where the Product Is/Will Be Offered

European Union

## Other Frameworks Evaluated

No alternatives exist.
Apple provides APIs that allow browsers to implement support for marketplace installation, but
this requires independent adoption by each browser and results in inconsistent behavior
across the ecosystem.
A system-level capability should not depend on third-party implementation to function. Only
iOS can ensure consistent invocation of the installation flow across all browsers and
applications.

## Confidential Treatment

Fully available to other developers

## Additional Information

Privacy Safeguards:
- This request does not involve access to user data.
- It only concerns the invocation of an existing system-level installation flow that is already
presented to users.
- No personal data, such as browsing information, is processed .
Technical Specification:
- iOS already recognizes marketplace installation links and invokes the installation flow in Safari
and system applications (Settings).
- The same capability can be exposed uniformly to all WebKit-based browsers and applications
- No additional data processing or collection is required
- Existing system security and confirmation capabilities remain unchanged
Success Criteria:
- Users can initiate marketplace installation from any browser or application without switching
to Safari
- Installation links behave consistently across Safari and all WebKit-based browsers
- In-app web views and embedded browsers can invoke the installation flow
- The system-level capability is documented and available to DMA-compliant alternative
marketplaces
- Behavior is auditable and consistent across devices

Thank you for your interoperability request.
It is currently in Phase I - Eligibility review. We will reach out via this forum if we need
clarification of your request.
If we do not reach out for clarification, we will endeavor to provide you with the outcome of the
initial assessment of our Phase I - Eligibility review within 20 working days of the date of your

May 4, 2026, from Apple to Developer:
We have reviewed your submission and determined that your request falls outside the scope of
Article 6(7) DMA. Specifically, your request asks Apple to create a new functionality on iOS that
would support the installation of alternative app marketplaces from web content independent
of web browsers which is likewise not available in the context of installation of the App Store
on iOS. For this reason, we are closing your request and not moving it to the next phase of our
interoperability process.
In relation to your specific request, Apple already supports the installation of alternative app
marketplaces using MarketplaceKit, including on Safari - for more information, see our
developer documentation available here: https://developer.apple.com/documentation/
marketplacekit.
As this determination is based on a non-technical consideration (i.e., whether Apple is legally
obligated to create new features within the meaning of Article 6(7) DMA), it is not eligible for the
dispute resolution mechanism.
This concludes our review under Article 6(7) DMA.
